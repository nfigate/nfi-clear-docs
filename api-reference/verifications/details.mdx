# Get Verification Details

Retrieve complete information about a verification, including:
- Full progress tracking
- Step history
- Access logs
- All submitted data
- Sub-step tracking for KYB

## Endpoint

```http
GET /api/v1/verifications/{id}/details
```

## Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| id | string | Yes | The verification ID |

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| X-API-Key | Yes | Your API key |

### Example Request

```bash
curl 'https://api.nfi-clear.com/api/v1/verifications/abc123def456/details' \
  -H 'X-API-Key: nfi_your_api_key_here'
```

## Response Fields

### Top-Level Fields

| Field | Type | Description |
|-------|------|-------------|
| id | string | Unique verification ID |
| type | string | `kyc` or `kyb` |
| subjectReference | string | Your reference ID |
| status | string | Current status |
| requestedAt | string | Creation timestamp |
| resolvedAt | string \| null | Resolution timestamp |
| resolutionNotes | string \| null | Notes if rejected/action needed |
| subscriptionName | string | Plan name |
| verificationUrl | string | Verification link |

### Progress Object

| Field | Type | Description |
|-------|------|-------------|
| currentStep | number | Current step index |
| stepHistory | array | Completed steps with timestamps |
| lastUpdated | string | Last progress update |
| currentAccountTypeStep | number | KYB: Business Details sub-step |
| currentDocumentStep | number | KYB: Document Upload sub-step |
| currentFinancialStep | number | KYB: Financial Info sub-step |
| currentConnectedPartiesStep | number | KYB: Connected Parties sub-step |
| currentCorporateGovernanceStep | number | KYB: Corporate Governance sub-step |

### Access Log Entry

| Field | Type | Description |
|-------|------|-------------|
| timestamp | string | Access timestamp |
| ip | string | Client IP address |
| device | string | Device type |
| browser | string | Browser info |
| os | string | Operating system |

### Submitted Data

#### KYC Data

```json
  "documentCountry": "Singapore",

  "frontImageUrl": "https://...",

  "selfieImageUrl": "https://...",
  "livenessImages": ["https://..."],

```

#### KYB Data

```json
  "connectedParties": { ... },
```

## Error Responses

### Not Found (404)

```json
{
  "success": false,
  "error": "Verification not found"
}
```

## Use Cases

This endpoint is ideal for:

- **Detailed review** - View all submitted information
- **Audit trails** - Access logs for compliance
- **Progress tracking** - Monitor user progress through KYB flow
- **Data export** - Extract verification data

## Performance Considerations

<Info>
  This endpoint returns a large payload. Use the standard [Get Verification](/api-reference/verifications/get) endpoint for simple status checks.
</Info>

## Data Retention

- Access logs: 90 days
- Submitted documents: 7 years (compliance)
- Verification records: Lifetime of relationship + 7 years
