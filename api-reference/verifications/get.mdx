# Get Verification

Retrieve the current status and basic information about a specific verification.

## Endpoint

```http
GET /api/v1/verifications/{id}
```

## Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| id | string | Yes | The verification ID |

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| X-API-Key | Yes | Your API key |

### Example Request

```bash
curl 'https://api.nfi-clear.com/api/v1/verifications/abc123def456' \
  -H 'X-API-Key: nfi_your_api_key_here'
```

## Response

### Success (200 OK)

```json
{
  "success": true,
  "data": {
    "id": "abc123def456",
    "type": "kyc",
    "subjectReference": "user-12345",
    "status": "submitted",
    "requestedAt": "2024-01-15T10:30:00.000Z",
    "resolvedAt": null,
    "currentStep": 4,
    "verificationUrl": "https://app.nfi-clear.com/verify/abc123def456",
    "submittedData": {
      "residenceCountry": "Singapore",
      "documentCountry": "Singapore",
      "documentType": "passport",
      "frontImageUrl": "https://...",
      "selfieImageUrl": "https://..."
    }
  }
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| id | string | Unique verification ID |
| type | string | `kyc` or `kyb` |
| subjectReference | string | Your reference ID |
| status | string | Current status |
| requestedAt | string | Creation timestamp |
| resolvedAt | string \| null | Resolution timestamp |
| currentStep | number | Current step in flow |
| verificationUrl | string | Verification link |
| submittedData | object \| undefined | Data for completed verifications |

## Status Values

| Status | Description |
|--------|-------------|
| `pending` | Verification created, user hasn't started |
| `submitted` | User completed the form, pending review |
| `action_needed` | Reviewer requested corrections |
| `approved` | Verification approved |
| `rejected` | Verification rejected |

## Error Responses

### Not Found (404)

```json
{
  "success": false,
  "error": "Verification not found"
}
```

<Warning>
  This endpoint returns a simplified view. For complete details including progress tracking and access logs, use the [Get Full Details](/api-reference/verifications/details) endpoint.

## Use Cases

This endpoint is ideal for:

- **Status polling** - Check if verification is complete
- **Dashboard display** - Show verification status to users
- **Status checks** - Before allowing user actions

## Polling Strategy

<Info>
  For real-time updates, use [webhooks](/webhooks) instead of polling.
</Info>

If you must poll, use an exponential backoff strategy:

```javascript
const checkStatus = async (verificationId, attempt = 0) => {
  const response = await fetch(
    `https://api.nfi-clear.com/api/v1/verifications/${verificationId}`,
    {
      headers: { 'X-API-Key': 'nfi_your_api_key' }
    }
  );
  
  const data = await response.json();
  
  if (data.data.status === 'approved' || data.data.status === 'rejected') {
    return data.data; // Final status reached
  }
  
  // Exponential backoff: 5s, 10s, 20s, 30s (max)
  const delay = Math.min(5000 * Math.pow(2, attempt), 30000);
  
  await new Promise(resolve => setTimeout(resolve, delay));
  return checkStatus(verificationId, attempt + 1);
};
```
