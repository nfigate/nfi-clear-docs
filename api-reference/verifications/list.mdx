# List Verifications

Retrieve a list of all your verifications with optional filtering and pagination.

## Endpoint

```http
GET /api/v1/verifications
```

## Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| status | string | No | Filter by status: `pending`, `submitted`, `action_needed`, `approved`, `rejected` |
| type | string | No | Filter by type: `kyc`, `kyb` |
| subjectReference | string | No | Search by subject reference (partial match) |
| page | number | No | Page number (default: 1) |
| limit | number | No | Items per page (default: 20, max: 100) |
| sortBy | string | No | Sort field: `requestedAt`, `status`, `subjectReference` (default: `requestedAt`) |
| sortOrder | string | No | Sort order: `asc`, `desc` (default: `desc`) |

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| X-API-Key | Yes | Your API key |

### Example Request

```bash
curl 'https://api.nfi-clear.com/api/v1/verifications?status=pending&type=kyc&page=1&limit=20' \
  -H 'X-API-Key: nfi_your_api_key_here'
```

### Response Fields

#### Verification Object

| Field | Type | Description |
|-------|------|-------------|
| id | string | Unique verification ID |
| type | string | `kyc` or `kyb` |
| subjectReference | string | Your reference ID |
| status | string | Current status |
| requestedAt | string | Creation timestamp (ISO 8601) |
| resolvedAt | string \| null | Resolution timestamp (if completed) |
| currentStep | number | Current step in flow |
| verificationUrl | string | Verification link |
| submittedData | object | Included for completed verifications |

#### Pagination Object

| Field | Type | Description |
|-------|------|-------------|
| page | number | Current page |
| limit | number | Items per page |
| totalCount | number | Total items matching filter |
| totalPages | number | Total pages available |
| hasMore | boolean | Whether more pages exist |

## Filter Examples

### Pending KYC Verifications

```bash
curl 'https://api.nfi-clear.com/api/v1/verifications?status=pending&type=kyc' \
  -H 'X-API-Key: nfi_your_api_key_here'
```

### Search by Subject Reference

```bash
curl 'https://api.nfi-clear.com/api/v1/verifications?subjectReference=user-123' \
  -H 'X-API-Key: nfi_your_api_key_here'
```

### Recently Approved (sorted by resolution date)

```bash
curl 'https://api.nfi-clear.com/api/v1/verifications?status=approved&sortBy=resolvedAt&sortOrder=desc' \
  -H 'X-API-Key: nfi_your_api_key_here'
```

### Pagination Example

```bash
# Page 2 with 50 items per page
curl 'https://api.nfi-clear.com/api/v1/verifications?page=2&limit=50' \
  -H 'X-API-Key: nfi_your_api_key_here'
```

## Implementation Tips

<Info>
  Use pagination to efficiently handle large datasets. The default limit is 20 items per page.
</Info>

### Auto-Pagination Example (JavaScript)

```javascript
async function getAllVerifications() {
  const allVerifications = [];
  let page = 1;
  let hasMore = true;

  while (hasMore) {
    const response = await fetch(
      `https://api.nfi-clear.com/api/v1/verifications?page=${page}&limit=100`,
      {
        headers: { 'X-API-Key': 'nfi_your_api_key' }
      }
    );
    
    const data = await response.json();
    allVerifications.push(...data.data.verifications);
    hasMore = data.data.pagination.hasMore;
    page++;
  }

  return allVerifications;
}
```
